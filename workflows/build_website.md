# Workflow: Build Business Website

## Objective
Generate a professional, responsive, single-page business website (self-contained `output/index.html`) from a plain-language business description. Three core steps are required; three optional enrichment steps add real reviews, a generated hero image, and the business's existing brand identity.

## Prerequisites
- Python 3.9+
- Dependencies installed: `pip install -r requirements.txt`
- `ANTHROPIC_API_KEY` set in `.env` (required)
- See optional steps below for their specific API keys

## Required Inputs
- A single free-text description of the business (1–5 sentences is sufficient)

## Expected Outputs
| File | Created by | Notes |
|------|-----------|-------|
| `.tmp/business_info.json` | Step 1 | Structured business data; enriched by optional steps |
| `.tmp/website_copy.json` | Step 2 | Marketing copy for all sections |
| `output/index.html` | Step 3 | Self-contained, opens in any browser |
| `output/assets/logo.png` | Step 1b (optional) | Brand logo downloaded from Brandfetch or scraped |
| `output/assets/hero_background.jpg` | Step 1c (optional) | AI-generated hero background image |

## Core Pipeline

### Step 1 — Gather Business Info
**Tool:** `tools/gather_business_info.py`

```bash
# Basic (existing behavior)
python tools/gather_business_info.py "Your business description here"

# With Google Places integration (pass place_id from qualified_leads.json)
python tools/gather_business_info.py "Your business description here" --place-id ChIJ...
```

**What it does:**
- Calls Claude (`claude-sonnet-4-6`) to extract: business name, tagline, industry, services (3–5), about copy, target audience, color scheme, contact placeholders, social links
- With `--place-id`: also calls Google Places API to fetch real rating, total review count, and top 5 customer reviews. Adds them to `business_info.json` under `google_places`

**Where to get the place_id:** Every entry in `.tmp/qualified_leads.json` (generated by `find_local_leads.py`) has a `place_id` field. Copy it from there.

**API cost (Places):** ~$0.017–0.02 per call (Advanced SKU). Covered by existing Google Cloud setup.

**Edge cases:**
- Short descriptions (1–3 words) → LLM infers heavily; review `.tmp/business_info.json` before proceeding
- Missing `ANTHROPIC_API_KEY` → script exits with clear error
- Missing `GOOGLE_PLACES_API_KEY` with `--place-id` → warning printed, Places enrichment skipped, core info still written

---

### Step 2 — Generate Website Copy
**Tool:** `tools/generate_copy.py`

```bash
python tools/generate_copy.py
```

**What it does:**
- Reads `.tmp/business_info.json`
- Calls Claude with a conversion copywriting prompt: benefit-driven headlines, trust signals, clear CTAs, no jargon
- Generates: hero, about (3 paragraphs), service cards, social proof stats, CTA banner, footer tagline
- Writes to `.tmp/website_copy.json`

---

### Step 3 — Build the Website
**Tool:** `tools/build_website.py`

```bash
python tools/build_website.py
```

**What it does:**
- Reads `.tmp/business_info.json` and `.tmp/website_copy.json`
- Generates `output/index.html` with Tailwind CSS (CDN), Google Fonts, and inline SVG icons
- **Multi-axis variant system**: uses a deterministic hash of the business name to assign from:
  - **5 font pairings** (Classic, Editorial, Refined, Geometric, Warm)
  - **4 page layouts** (Classic, Story-first, Services-focused, Minimal) — each reorders sections differently
  - **4 hero styles** (Centered, Left-aligned, Diagonal clip-path, Split two-column)
  - **4 service layouts** (Card grid, Alternating rows, Compact list, Featured + grid)
  - **5 visual personalities** (Minimal, Bold, Warm/Artisanal, Corporate, Modern) — each with distinct CSS
  - Total: **1,600 unique combinations**. Same business name always produces the same design
- **Textile-specific trust badges** that vary by business type (embroidery, home textiles, sewing, fashion, B2B)
- **Adapts automatically** to any optional enrichment data present in `business_info.json`:
  - `google_places` → renders a real Google review cards section + real rating in stats
  - `brand.logo_local_path` → shows brand logo image in nav instead of text
  - `brand.primary_color` → overrides Claude-generated primary color with real brand color
  - `hero_image_path` → uses AI-generated photo as hero background with dark overlay (instead of CSS gradient)
  - `hero_video_url` → uses mp4 video as hero background (autoplay, muted, loop). Takes priority over `hero_image_path` when both are set
  - `hero_video_poster` → optional static poster image displayed before the video loads; falls back to `hero_image_path` if not set
  - `business_type` → influences trust badge selection and, optionally, personality override
  - `personality` → if set (e.g., "bold", "minimal"), overrides the hash-based personality selection

**Output:** `output/index.html` (~50–120 KB, opens directly in any browser, no server needed)

**Design quality tip:** If the **UI UX Pro Max** skill set (nextlevelbuilder) is installed in Claude Code, invoke it before this step for premium clients. It gives Claude richer UI/UX design judgment — better layout hierarchy, spacing, and visual polish in the generated HTML. Use selectively for clients where presentation quality matters most (e.g., tech, hospitality, high-end services).

## Optional Enrichment Steps (run between Step 1 and Step 2)

### Step 1b — Extract Brand Identity (Logo + Colors)
**Tool:** `tools/extract_brand.py`

**New API key required:** `BRANDFETCH_API_KEY` in `.env`
**Sign up:** brandfetch.com/developers (free tier: 500 req/month)

```bash
# Domain auto-inferred from contact.email in business_info.json
python tools/extract_brand.py

# Or pass domain explicitly
python tools/extract_brand.py --domain example.com
```

**What it does:**
1. Calls Brandfetch API with the business's domain to get their official logo and brand hex colors
2. Downloads the logo to `output/assets/logo.{ext}`
3. Falls back to BeautifulSoup scraper if Brandfetch returns no result (scrapes favicon + dominant CSS hex from their existing site)
4. Writes `brand` key to `.tmp/business_info.json`

**Effect on final site:**
- Nav shows the real brand logo instead of text
- Primary color switches to the brand's actual hex (e.g., matching their existing site)

**Edge cases:**
- `BRANDFETCH_API_KEY` not set → skips Brandfetch, tries scraper fallback
- Business has no existing website → scraper will fail; `brand` key won't be written (site uses Claude-generated colors)
- Logo download fails → warning printed; `logo_local_path` absent; nav falls back to text

### Step 1c — Generate Hero Image
**Tool:** `tools/generate_images.py`
**Primary API key:** `GEMINI_API_KEY` in `.env` — Google AI Studio (free tier available; get key at aistudio.google.com)
**Fallback API key:** `STABILITY_API_KEY` in `.env` — platform.stability.ai (~$0.003/image, Stable Image Core)

```bash
python tools/generate_images.py
```

**What it does:**
1. Reads `industry` from `.tmp/business_info.json`
2. Selects an industry-specific photorealistic prompt (30+ industries mapped; fallback for others)
3. **Tries Gemini image generation first** (higher quality, 4K, superior text rendering)
4. **Falls back to Stability AI** if Gemini fails or key is missing
5. Saves image to `output/assets/<slug>_hero.jpg`
6. Writes `"hero_image_path": "assets/<slug>_hero.jpg"` into `business_info.json`

**Effect on final site:**
- Hero section uses the real photo as background with a semi-transparent dark overlay (ensures headline text stays readable)
- Without this step, the hero uses a professional CSS gradient (original behavior)

**Edge cases:**
- Neither key set → warning printed; `hero_image_path` not written; pipeline continues without image
- Gemini model names change frequently and may return 404 — the script tries multiple names and falls back to Stability AI automatically
- Stability AI valid aspect ratios: `16:9`, `3:2`, `1:1`, `4:5`, `2:3` — `4:3` is invalid and will error
- Stability API returns an error → error details printed; `hero_image_path` not written

---

### Step 1d — Generate Catalog Images (textile / product businesses)
**Tool:** `tools/generate_catalog_images.py`
**Same API keys as Step 1c** (Gemini primary, Stability AI fallback)

```bash
python tools/generate_catalog_images.py
```

**What it does:**
1. Generates 4 fabric/product category images using AI (Gemini → Stability AI fallback)
2. Saves to `output/assets/mubetex_catalog_moda.jpg`, `_hogar.jpg`, `_sostenible.jpg`, `_especial.jpg`
3. Already-existing images are skipped (idempotent)

**When to run:** Any time the website has a catalog or product grid section that benefits from real photographic imagery instead of color gradients. Edit prompts in the script to match the specific product categories.

---

### Step 1e — Find Hero Video (stock footage)
**Tool:** `tools/find_hero_video.py`
**New API key required:** `PEXELS_API_KEY` in `.env`
**Sign up:** pexels.com/api (completely free, 200 req/hour)

```bash
# Auto-selects query based on industry in business_info.json
python tools/find_hero_video.py

# Or override with a custom search query
python tools/find_hero_video.py --query "textile fabric weaving"
```

**What it does:**
1. Reads `industry` from `.tmp/business_info.json`
2. Maps the industry to an optimised video search query (30+ industries mapped; generic fallback for others)
3. Searches Pexels Video API for HD/4K landscape stock footage
4. **Smart selection**: filters by minimum 1920×1080, landscape orientation, 3–30s duration (ideal for looping)
5. Downloads the best match to `output/assets/<slug>_hero.mp4`
6. Downloads a poster thumbnail to `output/assets/<slug>_hero_poster.jpg`
7. Writes `hero_video_url` and `hero_video_poster` into `business_info.json`

**Effect on final site:**
- Hero section plays a full-width looping background video instead of a static image
- `hero_video_url` takes priority over `hero_image_path` when both are present
- Poster image shown before the video loads (better perceived performance)

**Edge cases:**
- `PEXELS_API_KEY` not set → error printed; script exits; pipeline continues without video
- No videos match quality criteria → warning printed; try `--query` with broader terms
- Downloaded file too small (< 500 KB) → treated as corrupted, removed automatically
- Both `hero_video_url` and `hero_image_path` set → video takes priority; image used as fallback poster

**When to run:** After Step 1c (hero image). The hero image serves as a fallback if video fails to load. Use `--query` to fine-tune relevance for niche industries not in the default mapping.

**Effect on final site:** Catalog section cards use the AI-generated photos as backgrounds with color-tinted overlays.

**Edge cases:**
- Image already exists → skipped (safe to re-run)
- API fails → card uses its fallback CSS gradient background

## Known Constraints / Lessons Learned
- Tailwind CDN is fine for single-site generation but shouldn't be used in production apps with many pages
- The contact form is static HTML — does not submit anywhere. Integrate Formspree or EmailJS to make it functional
- Color hex values must be valid 6-digit hex codes; `darken_color()` in `build_website.py` will use a fallback if invalid
- Brandfetch free tier: 500 requests/month. More than sufficient for lead-based outreach volume
- Gemini image generation model names change frequently and may return 404. The fallback to Stability AI is automatic and reliable — treat Stability AI as the de facto primary
- Stability AI valid aspect ratios: `16:9`, `3:2`, `1:1`, `4:5`, `2:3`, `9:16`, `9:21`, `21:9`, `5:4`. **`4:3` is invalid — always use `3:2` instead**
- Both image generators are non-deterministic — re-running produces a different image each time
- The `output/assets/` directory is created automatically by both `extract_brand.py` and `generate_images.py`
- When sending the demo to a lead, send the entire `output/` folder (not just `index.html`), as the HTML references `assets/` for logo and hero image

## Quality Rules — Learned from Production Errors

These rules are enforced in the code. This section documents the reasoning so they are never accidentally undone.

### Google Reviews
- **NEVER invent, translate, or complete review text.** Only render what is returned by Google Places API verbatim.
- If `review.text` is empty (the user left only a star rating), render the card without a quote paragraph — stars + name + date only. Do not fill in invented text.
- Keep the review in its original language. Do not translate.
- `build_reviews_html()` in `build_website.py` enforces this with: `if raw_text: render quote else: skip quote`

### WhatsApp vs Landline
- **Only use `wa.me/` links for mobile numbers.** In Spain: mobile numbers start with 6 or 7. Landlines start with 9 (e.g., 965..., 91..., 93...).
- Using WhatsApp for a landline creates a broken/dead link and looks unprofessional to the client.
- `is_spanish_mobile()` in `build_website.py` enforces this. If mobile → WhatsApp green button. If landline → regular `tel:` phone button.
- This logic is in `build_contact_info_html()`. Any custom floating button added to a website must follow the same rule.

### Scroll Indicator Animations + AOS
- The `.hero-scroll-indicator` bounce animation class must go on the **inner SVG element only**, not on the `<a>` wrapper.
- The `<a>` wrapper's parent div usually has `data-aos="fade-up"`. AOS uses `transform` for its animations; the bounce keyframe also uses `transform`. Putting both on the same element causes jitter/vibration.
- Use `animation-delay: 1.5s` so the bounce starts after AOS fade-in completes (~1s + buffer).
- Use `will-change: transform` for smooth GPU compositing.
